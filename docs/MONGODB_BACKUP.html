Sauvegarde MongoDB — Wise Eat

Scripts infra · VPS /opt/wise-eat · credentials /opt/wise-eat-api/.env.prod

Architecture

MongoDB rs0 → Dump local 03:30 → /var/backups/wise-eat-mongodb → Upload dim. 04:00 → GCS · Firebase · AWS

CLI : sudo ./scripts/mongodb-backup.sh <commande> · Doc Markdown : docs/MONGODB_BACKUP.md

Credentials (.env.prod)

Les uploads cloud n’utilisent pas de clés dupliquées dans .env.mongodb. Ils lisent le fichier API prod :

Variable .env.prodDestination backup
GCS_BUCKET / GOOGLE_CLOUD_STORAGE_BUCKETgs://{bucket}/mongodb/
AM_FIREBASE_STORAGE_BUCKETgs://{bucket}/mongodb/
AWS_S3_BUCKET + AWS_*s3://{bucket}/mongodb/
GOOGLE_APPLICATION_CREDENTIALS ou accounts.jsonAuth GCS / Firebase

Override chemin : MONGO_CLOUD_API_ENV=/opt/wise-eat-api/.env.prod dans mongodb/.env.mongodb.

Rotation cloud — 4 slots / mois

Chaque dimanche, une archive .tar.gz complète écrase le même nom chaque mois (économie d’espace).

Semaine (jour du mois)Fichier cloud
1 — jours 1 à 7Backup_DB_1.tar.gz
2 — jours 8 à 14Backup_DB_2.tar.gz
3 — jours 15 à 21Backup_DB_3.tar.gz
4 — jours 22 à finBackup_DB_4.tar.gz

Un fichier Backup_DB_N.meta.json (date, slot, base) est uploadé à côté.

Installation

cd /opt/wise-eat
sudo git pull

# mongodb/.env.mongodb :
#   MONGO_CLOUD_BACKUP_ENABLED=1
#   MONGO_CLOUD_API_ENV=/opt/wise-eat-api/.env.prod

sudo ./scripts/mongodb-backup.sh install-all

Prérequis : gcloud ou gsutil, aws CLI installés sur le VPS.

Exemples

# Vérifier la config avant premier upload
./scripts/mongodb-backup.sh env-check

# Test sans upload
sudo MONGO_CLOUD_BACKUP_FORCE=1 ./scripts/mongodb-backup.sh cloud-dry-run

# Upload réel (forcer un jour quelconque)
sudo MONGO_CLOUD_BACKUP_FORCE=1 ./scripts/mongodb-backup.sh cloud

# État global
./scripts/mongodb-backup.sh status

# Logs
sudo tail -50 /var/log/wise-eat-mongodb-backup.log
sudo tail -50 /var/log/wise-eat-mongodb-cloud-backup.log

Restauration depuis le cloud

AWS S3

aws s3 cp s3://VOTRE-BUCKET/mongodb/Backup_DB_2.tar.gz /tmp/
tar -xzf /tmp/Backup_DB_2.tar.gz -C /tmp/mongo-restore/
docker cp /tmp/mongo-restore/. wise-eat-mongo-1:/tmp/restore/
docker exec wise-eat-mongo-1 mongorestore \
  --username="$MONGO_ROOT_USER" --password="$MONGO_ROOT_PASSWORD" \
  --authenticationDatabase=admin --gzip --drop /tmp/restore/

GCS / Firebase Storage

export GOOGLE_APPLICATION_CREDENTIALS=/opt/wise-eat-api/accounts.json
gcloud storage cp gs://BUCKET/mongodb/Backup_DB_2.tar.gz /tmp/
# puis tar + docker cp + mongorestore comme ci-dessus

Attention --drop supprime les collections existantes avant restauration. Arrêter ou mettre en maintenance les apps écrivant en base.
